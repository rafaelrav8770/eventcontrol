1. Introducción

En el presente documento se describe la documentación técnica de la construcción del sistema EventControl, desarrollado durante la unidad de Ingeniería de Software. Este documento tiene como objetivo explicar el proceso de desarrollo del software, las tecnologías utilizadas, la estructura del sistema y su funcionamiento general.

EventControl es un sistema web diseñado para la administración integral de eventos, permitiendo gestionar invitados, asignar mesas, controlar el acceso mediante códigos QR y llevar un registro digital de la asistencia. El sistema fue desarrollado con tecnologías web modernas y se encuentra desplegado en la nube.

El desarrollo de software es una disciplina fundamental dentro de la ingeniería de sistemas, ya que permite transformar problemáticas reales en soluciones digitales eficientes y escalables. A través de la construcción de este sistema, se aplicaron principios de diseño, programación, bases de datos y arquitectura de software.

El lector encontrará dentro de este documento la justificación que motivó la creación del sistema, una descripción general de sus funcionalidades, las tecnologías empleadas, la estructura de módulos y base de datos, el flujo de funcionamiento, el proceso de desarrollo seguido, evidencias del trabajo realizado y las conclusiones obtenidas.

2. Justificación del sistema

El desarrollo de EventControl surge a partir de la necesidad de mejorar la organización y administración de eventos sociales, especialmente en lo que respecta al control de invitados, la asignación de mesas y el registro de acceso. En muchos eventos, estos procesos se realizan de forma manual mediante listas impresas, lo cual puede generar errores, pérdida de información, duplicidad de registros o dificultades para controlar la entrada de los asistentes.

En el caso específico de este proyecto, se identificó la problemática de organizar un evento con un gran número de invitados donde era indispensable contar con un sistema que permitiera:

  • Registrar digitalmente a los invitados y asignarles una mesa específica.
  • Generar códigos de acceso únicos para cada familia o grupo de invitados.
  • Permitir que los invitados confirmen su asistencia de forma digital.
  • Controlar el ingreso de los asistentes el día del evento mediante escaneo de códigos QR.
  • Visualizar en tiempo real estadísticas de asistencia y ocupación de mesas.

Por esta razón se decidió desarrollar un sistema web que permitiera gestionar de manera digital toda la información relacionada con el evento y sus invitados. De esta manera se busca optimizar la organización, facilitar el acceso a la información en tiempo real y mejorar el control de asistencia durante el evento.

Además, el desarrollo de este sistema permite aplicar los conocimientos adquiridos durante la asignatura, tales como el uso de lenguajes de programación, manejo de bases de datos, principios de desarrollo de software, arquitectura cliente-servidor y despliegue en la nube.

3. Descripción general del sistema

Nombre del sistema EventControl
Tipo de aplicación Aplicación web (Frontend + Backend como servicio)
Despliegue Vercel (hosting estático en la nube)

Objetivo del sistema

Desarrollar un sistema web que permita administrar eventos, gestionar invitados, asignar mesas, generar códigos de acceso únicos y controlar el ingreso de los asistentes mediante escaneo de códigos QR, todo en tiempo real.

Problema que resuelve

EventControl resuelve la problemática de la organización manual de eventos con gran cantidad de invitados, eliminando el uso de listas impresas, reduciendo errores humanos en el conteo de asistentes y proporcionando estadísticas de asistencia en tiempo real a los organizadores.

Usuarios del sistema

Rol de usuario Descripción

5. Arquitectura o estructura del sistema

EventControl utiliza una arquitectura cliente-servidor donde el frontend se ejecuta completamente en el navegador del usuario y se comunica directamente con Supabase (Backend-as-a-Service) para las operaciones de datos. El sistema no requiere un servidor backend tradicional; toda la lógica de negocio reside en el cliente y la seguridad se garantiza mediante las políticas de Row Level Security (RLS) de Supabase.

A continuación se describe cada módulo:

1. Página de Invitación Pública

Es la página principal del sistema, accesible al público. Presenta la información del evento con un diseño elegante que incluye: pantalla de bienvenida con animación, nombres de los organizadores, cuenta regresiva al evento, información de las familias, detalles del evento (fecha, hora, ubicación), itinerario, galería de fotos, y un botón para confirmar asistencia. Incluye un menú de navegación flotante tipo hamburguesa y un lightbox para las fotos.

2. Panel Administrativo (Dashboard)

Módulo protegido por autenticación que permite a los administradores del evento gestionar toda la información:

  • Gestión de mesas: Crear mesas individuales o masivas, definir capacidad, eliminar mesas vacías y visualizar ocupación en un grid visual con barras de progreso.
  • Gestión de pases de invitados: Crear pases con nombre de familia, teléfono, número de invitados y mesa asignada. Cada pase genera automáticamente un código único de 4 caracteres.
  • Estadísticas en tiempo real: Dashboard con contadores de mesas totales, invitados totales, invitados confirmados y pendientes.
  • Filtros y búsqueda: Filtrar invitados por estado (todos, confirmados, pendientes, ingresados) y por creador (novio/novia). Búsqueda por nombre de familia.
  • Acciones sobre pases: Editar, eliminar y copiar códigos al portapapeles.

3. Control de Acceso / Escáner QR

Módulo diseñado para ser utilizado en dispositivos móviles durante el evento. Permite al personal de acceso:

  • Escanear códigos QR de las invitaciones digitales usando la cámara del dispositivo.
  • Buscar invitados manualmente ingresando su código de 4 caracteres.
  • Verificar si el invitado está confirmado y si aún no ha ingresado.
  • Registrar la entrada seleccionando cuántas personas del pase ingresan.
  • Visualizar estadísticas de ingreso (esperados, adentro, pendientes).
  • Ver un log de las últimas entradas registradas y la lista de pases pendientes.
  • Recibir retroalimentación sonora (beep de éxito o error) al escanear.

4. Confirmación de Asistencia

Página pública donde los invitados pueden ingresar su código de acceso de 4 dígitos para confirmar su asistencia al evento y descargar su invitación digital con código QR.

5. Sistema de Autenticación

Los módulos administrativo y de control de acceso están protegidos por un sistema de login que utiliza la autenticación de Supabase. Solo los usuarios registrados con los roles adecuados (admin, groom, bride, access_control) pueden acceder a estas funcionalidades.

6. Base de datos del sistema

El sistema utiliza Supabase como servicio de base de datos, el cual proporciona una base de datos PostgreSQL en la nube con soporte para Row Level Security (RLS), autenticación integrada y suscripciones en tiempo real. A continuación se describen las tablas principales del sistema:

Tablas principales

Tabla Descripción
configuracion_evento Almacena la información general del evento: número total de mesas y capacidad por mesa por defecto.
mesas Contiene el registro de cada mesa del evento con su número, capacidad máxima y número de asientos ocupados.
pases_invitados Tabla principal del sistema. Registra cada pase de invitado con: nombre de familia, teléfono, total de invitados, código de acceso único de 4 caracteres, mesa asignada, estado de confirmación e invitados ingresados.
registros_entrada Registra cada evento de ingreso al evento: qué pase fue verificado, cuántos invitados ingresaron y quién del staff los verificó.
descargas_invitacion Guarda un registro estadístico de cada vez que un invitado descarga su invitación digital.
perfiles_usuario Almacena información adicional de los usuarios del sistema (administradores, staff): nombre, correo y rol asignado.

Seguridad de la base de datos (RLS)

Todas las tablas tienen habilitado Row Level Security (RLS), lo cual permite definir políticas de acceso a nivel de fila. Las políticas implementadas son:

  • Usuarios autenticados: Pueden leer, insertar, actualizar y eliminar registros en todas las tablas (según su rol).
  • Usuarios anónimos (invitados): Pueden leer los pases de invitados y la configuración del evento para la página de confirmación. También pueden actualizar el estado de confirmación de un pase y registrar descargas de invitaciones.

7. Funcionamiento del sistema

A continuación se describe el flujo de funcionamiento del sistema paso a paso, desde la preparación del evento hasta el registro de asistencia:

Configuración inicial del evento. El administrador inicia sesión en el panel administrativo y configura los parámetros generales del evento: cantidad de mesas y capacidad por mesa.

Creación de mesas. Se crean las mesas disponibles para el evento. Pueden crearse de forma individual (especificando capacidad) o de forma masiva (varias mesas con la misma capacidad). El sistema muestra un grid visual con la ocupación de cada mesa.

Registro de invitados. El administrador registra cada grupo de invitados creando un "pase" con los datos: nombre de la familia, teléfono, número de personas y mesa asignada.

Generación de código de acceso. El sistema genera automáticamente un código único de 4 caracteres alfanuméricos para cada pase. Este código se utiliza como identificador del invitado y se puede copiar al portapapeles para compartirlo.

Distribución de invitaciones. El organizador comparte el código de acceso con los invitados. Estos pueden ingresar a la página de confirmación del sistema e introducir su código para descargar su invitación digital con código QR.

Confirmación de asistencia. Los invitados acceden a la página de confirmación, ingresan su código de 4 dígitos y confirman su asistencia. El sistema actualiza el estado del pase en la base de datos y queda reflejado en el panel administrativo en tiempo real.

Control de acceso durante el evento. El día del evento, el personal de control de acceso utiliza el módulo de escáner desde su dispositivo móvil. Al escanear el código QR del invitado (o buscarlo manualmente), el sistema verifica su pase, muestra su información y permite registrar cuántas personas del grupo ingresan.

Registro de asistencia. Cada ingreso queda registrado en la base de datos con la hora exacta y el staff que lo verificó. El sistema actualiza las estadísticas en tiempo real: invitados esperados, invitados adentro y pendientes por llegar.

Nota: El módulo de control de acceso utiliza suscripciones en tiempo real de Supabase, por lo que cualquier cambio realizado desde el panel administrativo (como la creación o edición de un pase) se refleja automáticamente en el escáner sin necesidad de recargar la página.

8. Proceso de desarrollo del sistema

El desarrollo de EventControl se llevó a cabo siguiendo un proceso estructurado en varias etapas, desde el análisis inicial de la problemática hasta las pruebas finales del sistema:

Etapa 1 — Análisis del sistema

Se identificó la problemática principal: la necesidad de un sistema digital para gestionar invitados en eventos con gran número de asistentes. Se definieron los requerimientos funcionales del sistema: registro de invitados, asignación de mesas, generación de códigos de acceso, confirmación de asistencia y control de ingreso con QR. También se definieron los roles de usuario necesarios (administrador, staff de acceso e invitado).

Etapa 2 — Diseño del sistema

Se diseñó la arquitectura general del sistema como una aplicación web estática con Supabase como backend. Se definió la estructura de la base de datos con sus 6 tablas principales y las relaciones entre ellas. Se planificó la división en módulos: página de invitación, panel administrativo, control de acceso y confirmación. Se seleccionaron las tecnologías a utilizar y se diseñaron los prototipos de la interfaz de usuario.

Etapa 3 — Desarrollo del frontend

Se programaron los módulos del sistema utilizando HTML, CSS y JavaScript. Se desarrolló la página de invitación con diseño responsivo, animaciones de scroll, cuenta regresiva, galería con lightbox y formulario de RSVP. Se construyó el panel administrativo con todas las funcionalidades de gestión de mesas e invitados, incluyendo filtros, búsqueda y estadísticas. Se implementó el módulo de control de acceso con integración de la cámara para escaneo QR, retroalimentación sonora y registro de entradas.

Etapa 4 — Implementación de la base de datos

Se creó la estructura completa de la base de datos en Supabase mediante un script SQL (crear-base-datos.sql) que incluye la creación de las 6 tablas, índices para optimización de consultas y las políticas de seguridad RLS para controlar el acceso a los datos según el nivel de autenticación del usuario. Se desarrollaron scripts en Python para la creación masiva de usuarios del sistema.

Etapa 5 — Integración y despliegue

Se integró el frontend con la base de datos de Supabase mediante el SDK de JavaScript. Se configuró el despliegue automático en Vercel con integración a GitHub, permitiendo que cada cambio en el repositorio se refleje automáticamente en producción. Se configuró la generación dinámica de variables de entorno mediante un script de build (generate-env.js).

Etapa 6 — Pruebas del sistema

Se realizaron pruebas funcionales de cada módulo para verificar el correcto funcionamiento del sistema: creación y eliminación de mesas, creación de pases con códigos únicos, confirmación de asistencia desde la página pública, escaneo de códigos QR desde dispositivos móviles, verificación del registro de entradas en tiempo real y validación de las políticas de seguridad de la base de datos.

9.2 Código de la base de datos

CREATE TABLE pases_invitados (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    nombre_familia VARCHAR(255) NOT NULL,
    telefono VARCHAR(20),
    total_invitados INTEGER NOT NULL DEFAULT 2,
    codigo_acceso CHAR(4) NOT NULL UNIQUE,
    mesa_id UUID REFERENCES mesas(id),
    confirmado BOOLEAN DEFAULT FALSE,
    confirmado_en TIMESTAMP WITH TIME ZONE,
    invitados_ingresados INTEGER DEFAULT 0,
    todos_ingresaron BOOLEAN DEFAULT FALSE,
    creado_por UUID REFERENCES auth.users(id),
    creado_en TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    actualizado_en TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

10. Conclusión

El desarrollo del sistema EventControl permitió aplicar de manera práctica los conocimientos adquiridos durante la asignatura de Ingeniería de Software, abarcando áreas fundamentales como la programación web, el diseño de bases de datos relacionales, la arquitectura cliente-servidor y el despliegue de aplicaciones en la nube.

A través de este proceso se logró crear una solución funcional y completa que aborda la problemática de la organización manual de eventos con gran cantidad de invitados. El sistema permite gestionar digitalmente toda la información relacionada con los invitados, las mesas y el acceso al evento, proporcionando herramientas que facilitan la labor tanto de los organizadores como del personal de control de acceso.

Entre los logros más destacados del desarrollo se encuentran:

  • La implementación de un sistema de códigos de acceso únicos que simplifican la identificación de invitados.
  • La integración de un escáner de códigos QR que permite verificar el acceso de forma rápida y precisa.
  • El uso de una base de datos en la nube con suscripciones en tiempo real que mantiene la información actualizada en todos los dispositivos simultáneamente.
  • El despliegue exitoso del sistema en una plataforma en la nube (Vercel), haciéndolo accesible desde cualquier dispositivo con conexión a internet.
  • La implementación de políticas de seguridad a nivel de base de datos (RLS) que protegen la información sensible del sistema.

Además, esta actividad permitió comprender de manera práctica el proceso completo de desarrollo de un sistema de software dentro del área de ingeniería, desde la identificación de la problemática y el diseño de la solución hasta la implementación, las pruebas y el despliegue en producción. Estas competencias son fundamentales para la formación profesional en el área de ingeniería de sistemas y desarrollo de software.

En conclusión, EventControl demuestra que es posible desarrollar soluciones tecnológicas eficientes para problemas cotidianos utilizando herramientas modernas de desarrollo web, y que el proceso de construcción de software requiere de un enfoque estructurado que integre análisis, diseño, desarrollo, pruebas y despliegue de manera ordenada y documentada.
