1. 🚪 Pantalla de Entrada (main.js)

¿Qué hace?

Es la primera pantalla que ve el invitado. Muestra las iniciales "A | B", la fecha, y un botón "ENTRAR". Opcionalmente muestra el nombre de la familia si la URL tiene el parámetro ?pase=NombreFamilia.

Funciones implementadas:

initGuestPass() — Leer parámetro de URL

function initGuestPass() {
    const urlParams = new URLSearchParams(window.location.search);
    const pase = urlParams.get('pase');
    if (pase && guestPass) {
        guestPass.textContent = `Pase para: ${decodeURIComponent(pase.replace(/\+/g, ' '))}`;
    }
}
  • Qué hace: Lee el parámetro ?pase= de la URL usando la API URLSearchParams
  • Para qué sirve: Cuando los novios comparten un link personalizado como miboda.com?pase=Familia+Garcia, el invitado ve su nombre en la pantalla de entrada
  • Conectado a: El elemento HTML #guest-pass dentro de la sección .entry-screen

initEntryScreen() — Transición de entrada

function initEntryScreen() {
    enterBtn.addEventListener('click', () => {
        entryScreen.classList.add('fade-out');    // animacion CSS de 0.8s
        setTimeout(() => {
            entryScreen.style.display = 'none';
            mainContent.classList.remove('hidden');
            requestAnimationFrame(() => {
                mainContent.classList.add('show');
                checkScrollAnimations();
            });
        }, 800);
    });
}
  • Qué hace: Al hacer click en "ENTRAR", agrega la clase fade-out que tiene una transición CSS de opacity y visibility de 0.8 segundos. Después oculta la pantalla de entrada y muestra el contenido principal.
  • Implementación técnica: Usa requestAnimationFrame() para asegurar que el navegador renderize el cambio antes de activar las animaciones de scroll.
  • Conectado a: CSS (.entry-screen.fade-out { opacity: 0; visibility: hidden; }), #main-content que inicia con clase hidden

2. 🧭 Navegación Flotante (main.js)

¿Qué hace?

Muestra un botón de menú hamburguesa que aparece solo después de scrollear pasado el hero. Al hacer click, abre un panel lateral con links a cada sección.

initFloatingNav() — Lógica del menú

window.addEventListener('scroll', () => {
    if (currentScrollY > heroHeight - 100) {
        menuToggle.classList.add('visible');
    } else {
        menuToggle.classList.remove('visible');
        closeNav();
    }
}, { passive: true });
  • Qué hace: Escucha el evento scroll y compara la posición actual con la altura del hero. Si pasamos el hero, muestra el botón de menú.

  • Smooth scroll: Los links del menú usan window.scrollTo({ behavior: 'smooth' }) para animar el desplazamiento

3. ⏱️ Cuenta Regresiva (countdown.js)

¿Qué hace?

Calcula cuántos días, horas, minutos y segundos faltan para la boda y actualiza el DOM cada segundo.

const targetDate = new Date('2026-03-28T17:00:00-06:00');

function updateCountdown() {
    const now = new Date();
    const difference = targetDate - now;

    if (difference <= 0) {
        // Cambia el título a "¡Es Hoy!"
        return;
    }

    const days = Math.floor(difference / (1000 * 60 * 60 * 24));
    const hours = Math.floor((difference % (1000 * 60 * 60 * 24)) / (1000 * 60 * 60));
    // ... minutos y segundos igual

    daysEl.textContent = days.toString();
    hoursEl.textContent = hours.toString().padStart(2, '0');
}

updateCountdown();              // primera vez
setInterval(updateCountdown, 1000);  // cada segundo
  • Fecha objetivo: 2026-03-28T17:00:00-06:00 — 28 de marzo 2026, 5pm hora de México (UTC-6)
  • Cálculo: Resta la fecha actual de la fecha objetivo en milisegundos, luego divide entre 1000, 60, 3600 y 86400 para obtener días/horas/minutos/segundos
  • padStart(2, '0'): Agrega un cero a la izquierda cuando el número es de un solo dígito (ej: 5 → 05)
  • setInterval(fn, 1000): Ejecuta la función cada 1000 milisegundos (1 segundo)
  • Caso especial: Si difference <= 0 (ya pasó la fecha), pone todo en cero y cambia el título de la sección a "¡Es Hoy!"
  • Conectado a: Elementos #days, #hours, #minutes, #seconds en el HTML

4. 📸 Galería con Lightbox (gallery.js)

¿Qué hace?

La galería muestra un preview de 4 fotos (grid 2x2). Al hacer click en cualquiera, se abre un visor fullscreen (lightbox) con navegación por flechas, teclado y swipe en móvil.

Funciones principales:

Función Qué hace
initGallery() Recorre todos los .gallery-item, guarda sus URLs en un array, y les agrega un event listener de click
openLightbox(index) Abre el lightbox en la foto indicada, bloquea scroll del fondo con overflow: hidden
closeLightbox() Oculta el lightbox y restaura el scroll
updateLightboxImage() Cambia la imagen con una transición de opacidad (fade) de 150ms
prevImage() / nextImage() Navega circularmente: (currentIndex + 1) % images.length
handleSwipe() Detecta gestos táctiles con un umbral de 50px

Soporte de Swipe en Móvil:

lightbox.addEventListener('touchstart', (e) => {
    touchStartX = e.changedTouches[0].screenX;
}, { passive: true });

lightbox.addEventListener('touchend', (e) => {
    touchEndX = e.changedTouches[0].screenX;
    handleSwipe();
});

function handleSwipe() {
    const diff = touchStartX - touchEndX;
    if (Math.abs(diff) > 50) {  // 50px mínimo para swipe
        diff > 0 ? nextImage() : prevImage();
    }
}
  • Conectado a: Eventos touchstart/touchend del API de Touch Events
  • Atajos de teclado: Escape cierra, ArrowLeft/ArrowRight navegan

6. 🎭 Animaciones de Scroll (main.js)

¿Qué hace?

Los elementos aparecen con una animación suave (fade + slide upward) cuando el usuario hace scroll y los elementos entran en el viewport.

function initScrollAnimations() {
    const observer = new IntersectionObserver((entries) => {
        entries.forEach(entry => {
            if (entry.isIntersecting) {
                entry.target.classList.add('visible');
            }
        });
    }, {
        rootMargin: '0px 0px -50px 0px',  // se activa 50px antes del borde
        threshold: 0.1                     // con 10% visible ya anima
    });

    animatedElements.forEach(el => observer.observe(el));
}
  • IntersectionObserver: API moderna del navegador que detecta cuándo un elemento entra en la pantalla, es más eficiente que escuchar el evento scroll
  • rootMargin: -50px: La animación se activa cuando el elemento está a 50px del borde inferior de la pantalla
  • threshold: 0.1: Con solo el 10% del elemento visible, ya se activa la animación
  • CSS asociado: Los elementos inician con opacity: 0; transform: translateY(30px) y al tener la clase visible hacen transición a opacity: 1; transform: translateY(0)

7. 🖼️ Lazy Loading de Imágenes (main.js)

¿Qué hace?

Las imágenes no se cargan todas al inicio — solo se cargan cuando el usuario se acerca a ellas. Esto hace que la página cargue más rápido.

function initLazyLoading() {
    const imageObserver = new IntersectionObserver((entries) => {
        entries.forEach(entry => {
            if (entry.isIntersecting) {
                const img = entry.target;
                if (img.dataset.src) {
                    img.src = img.dataset.src;  // carga la imagen real
                    img.removeAttribute('data-src');
                }
                imageObserver.unobserve(img);  // deja de observar
            }
        });
    });
}
  • Técnica: Las imágenes en el HTML tienen data-src="foto.jpg" en vez de src. Cuando entran en viewport, se mueve el valor a src y la imagen se descarga.
  • unobserve(): Una vez cargada, dejamos de observar esa imagen para no gastar recursos.

function generateCode() {
    const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789';
    // Excluye I, O, 0, 1 para evitar confusión visual
    let code = '';
    for (let i = 0; i < 4; i++) {
        code += chars.charAt(Math.floor(Math.random() * chars.length));
    }
    return code;
}

9.4 Sistema de Filtros y Búsqueda

function filterGuests(filter) {
    const cards = document.querySelectorAll('[data-pass-id]');
    cards.forEach(card => {
        const status = card.dataset.status;
        const visible = filter === 'all' || status === filter;
        card.style.display = visible ? '' : 'none';
    });
}

function filterByCreator(creatorRole) {
    // Filtra pases por quién los creó (novio o novia)
    cards.forEach(card => {
        const creator = card.dataset.creator;
        card.style.display = (creator === creatorRole || !creatorRole) ? '' : 'none';
    });
}

function searchGuests(query) {
    // Búsqueda en tiempo real por nombre de familia
    cards.forEach(card => {
        const name = card.dataset.familyName.toLowerCase();
        card.style.display = name.includes(query.toLowerCase()) ? '' : 'none';
    });
}
  • 3 tipos de filtro: Por estado (todos, confirmados, adentro, pendientes), por creador (novio/novia), por nombre (búsqueda libre)
  • Conectado a: Atributos data-* del HTML que se generan dinámicamente al renderizar las cards

10. 📱 Escáner QR — Control de Acceso (scanner.js)

¿Qué hace?

Permite al personal de recepción escanear códigos QR de los invitados para registrar su entrada. Incluye retroalimentación auditiva, estadísticas en tiempo real, y búsqueda manual.

10.1 Inicialización

async function initScanner() {
    supabaseInstance = window.supabaseClient;

    // 1. Verificar autenticación
    const { data: { session } } = await supabaseInstance.auth.getSession();
    if (!session) { window.location.href = '/admin/index.html'; return; }

    // 2. Cargar datos de pases
    await loadData();

    // 3. Suscribirse a cambios en tiempo real
    initRealtime();

    // 4. Iniciar cámara QR
    startScanner();
}

10.2 Escáner de Cámara QR

function startScanner() {
    scanner = new Html5Qrcode("qr-reader");

    scanner.start(
        { facingMode: "environment" },  // cámara trasera del teléfono
        { fps: 10, qrbox: { width: 250, height: 250 } },
        onScanSuccess,  // callback cuando lee un QR
        onScanFailure   // callback silencioso cuando no hay QR
    );
}
  • Librería: Html5Qrcode — lee QR desde la cámara usando la API getUserMedia
  • facingMode: "environment": Usa la cámara trasera (la que apunta al frente) del teléfono
  • 10 FPS: Revisa la cámara 10 veces por segundo buscando un código QR

10.3 Procesamiento de Códigos

async function processCode(code) {
    // Busca el pase en la lista local (no hace query cada vez)
    const pass = guestPasses.find(p => p.codigo_acceso === code);

    if (!pass) {
        showGuestNotFound();  // código no existe
        playBeep('error');     // sonido de error
        return;
    }

    if (pass.todos_ingresaron) {
        showAlreadyUsed(pass); // ya entraron todos
        playBeep('error');
        return;
    }

    showGuestInfo(pass);   // muestra nombre, mesa, cuántos faltan
    playBeep('success');    // sonido de éxito
}
  • Búsqueda local: guestPasses.find() busca en la lista ya cargada en memoria, no hace una consulta a la base de datos cada vez que se escanea — esto es para velocidad.
  • Debounce: Si se escanea el mismo código dos veces seguidas, se ignora por 3 segundos.

10.4 Audio con Web Audio API

function playBeep(type = 'success') {
    const oscillator = audioContext.createOscillator();
    const gainNode = audioContext.createGain();

    oscillator.connect(gainNode);
    gainNode.connect(audioContext.destination);

    if (type === 'success') {
        oscillator.type = 'sine';                    // onda sinusoidal = sonido limpio
        oscillator.frequency.setValueAtTime(800, ...);  // 800Hz → 1200Hz (subida)
    } else {
        oscillator.type = 'sawtooth';                // onda diente de sierra = sonido áspero
        oscillator.frequency.setValueAtTime(200, ...);  // 200Hz → 150Hz (bajada)
    }
}
  • Web Audio API: Genera sonidos programáticamente sin necesidad de archivos de audio
  • Éxito: Tono ascendente (800→1200Hz) en onda sinusoidal, duración 0.1s
  • Error: Tono descendente (200→150Hz) en onda sawtooth, duración 0.3s
  • gainNode: Controla el volumen, haciendo un fade-out suave al final

10.6 Tiempo Real (Realtime)

function initRealtime() {
    supabaseInstance.channel('scanner_updates')
        .on('postgres_changes',
            { event: '*', schema: 'public', table: 'pases_invitados' },
            payload => {
                loadData();  // recarga todos los datos
            }
        )
        .subscribe();
}
  • WebSocket: Supabase Realtime abre una conexión WebSocket permanente con la base de datos
  • event: '*': Escucha cualquier cambio (INSERT, UPDATE, DELETE) en la tabla pases_invitados
  • ¿Para qué? Si los novios editan un pase desde el dashboard mientras la recepción está escaneando, los datos se actualizan automáticamente sin recargar la página

11. 🔗 Conexión a Supabase (supabase-config.js + env.js)

¿Qué hace?

Crea una sola instancia del cliente de Supabase y la comparte con todos los scripts.

// env.js — inyecta las credenciales como variable global
window.ENV = {
    SUPABASE_URL: "https://xethjgzynlkrwsirrzsf.supabase.co",
    SUPABASE_ANON_KEY: "eyJ..."
};

// supabase-config.js — crea el cliente
const env = window.ENV || {};
if (!window.supabaseClient && window.supabase) {
    window.supabaseClient = window.supabase.createClient(
        env.SUPABASE_URL, 
        env.SUPABASE_ANON_KEY
    );
}
  • Patrón Singleton: Se crea el cliente una sola vez en window.supabaseClient, y todos los scripts acceden a esa misma instancia
  • SUPABASE_ANON_KEY: Es la clave pública (anon key) que permite operaciones limitadas por Row Level Security (RLS)
  • Orden de carga en HTML: Siempre va env.js → supabase CDN → supabase-config.js → scripts de la página
